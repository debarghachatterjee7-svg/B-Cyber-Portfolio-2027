# 👀 Observations

While experimenting with Linux permissions, I observed that changing permission values immediately affected how Linux displayed access rights through the `ls -l` command. I noticed that `chmod 600` removed access from group members and other users, making the file accessible only to the owner.

During the SSH authentication attempt, I also observed that authentication involves much more than simply providing a password or private key. Debugging output demonstrated that SSH performs several verification steps before allowing access to a remote system.

---

# 🌍 Real World Application

Linux file permissions form one of the most important layers of system security. Incorrect permissions on SSH private keys may expose sensitive credentials or prevent successful authentication.

System administrators use permission management every day to secure:

- SSH private keys
- Configuration files
- Password databases
- Server certificates
- Backup files
- User home directories

Understanding permission management is therefore an essential skill for Linux administration and cybersecurity.
