# 📖 Theory Learned

Today's lesson introduced an entirely new category of Linux concepts.

I learned that computers ultimately store data in binary, but because binary is difficult for humans to read, hexadecimal is commonly used as a compact representation. Every hexadecimal digit represents four binary bits, making hexadecimal extremely useful when inspecting binary files.

I learned how the **xxd** utility converts binary files into hexadecimal and how the reverse operation reconstructs the original binary data.

Another major concept was Linux file identification. Rather than trusting a filename extension, Linux utilities inspect the internal file signature (magic bytes) to determine the real file type. This makes the **file** command one of the most useful investigation tools available.

The lesson also introduced Linux compression and archive utilities:

- gzip
- gunzip
- bzip2
- bunzip2
- tar

Instead of thinking of them as isolated commands, I learned that they form part of a complete workflow for handling unknown files.

---

# 💻 Practical Work

After completing the theory, I applied each concept through practical exercises.

I first experimented with hexadecimal files and practiced converting hexadecimal back into binary using the `xxd -r` command.

Next, I investigated unknown files using the `file` command before attempting any extraction. This reinforced the habit of identifying a file before deciding which utility should be used.

The practical then moved through several archive formats including gzip, bzip2 and tar. Multiple compressed layers had to be extracted one after another until the original ASCII text containing the password was finally recovered.

This exercise demonstrated that Linux investigation is often a repeated cycle of identifying, extracting and verifying files rather than solving the problem with a single command.
