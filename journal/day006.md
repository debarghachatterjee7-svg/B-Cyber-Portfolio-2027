## Biggest Learning

Linux commands follow a very strict syntax. Even one missing space changes how the shell interprets the command.

I also learned that:

- `find` searches for files.
- `grep` searches inside files.

---

## Biggest Confusion

Initially I typed:

```bash
grep linux.txt
```

instead of

```bash
grep Linux linux.txt
```

I learned that grep expects:

```
grep <pattern> <filename>
```

I also forgot spaces while using find:

```bash
find . -name*.txt
```

instead of

```bash
find . -name "*.txt"
```

This helped me understand how the shell parses commands.
