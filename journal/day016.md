# 💡 What I Learned

One of the biggest lessons today was that SSH is much more than just a command used to connect to another machine. It is a complete secure communication protocol designed to protect authentication and transmitted data using cryptographic techniques.

I learned that authentication can happen either through passwords or through SSH keys. Password authentication requires the server to verify a password, whereas key-based authentication uses a mathematically related pair of keys where only the public key is stored on the server while the private key remains safely on the client's computer.

I also understood that the private key must never be shared with anyone because possession of the private key is equivalent to possessing the user's identity.

# ❌ Mistakes Made Today

I accidentally saved my SSH key with a custom filename during my first attempt instead of accepting the default save location. Because of this, I became confused when trying to locate the generated key inside the `.ssh` directory.

Initially I also assumed that Linux would automatically create an `id_rsa` file, but I later learned that modern versions of OpenSSH generate Ed25519 keys by default unless another algorithm is explicitly specified.

I also noticed that I was assuming numeric permissions were directly shown by Linux, whereas Linux actually displayed symbolic permissions. I later learned how to convert symbolic permissions into their numeric equivalents.

These mistakes helped me understand Linux behavior much better than simply following instructions.
