# 🔍 Biggest Learning

Linux interprets filenames beginning with `-` as command-line options. Using `--` tells Linux that everything following it is a filename, while `./` forces Linux to search for the file in the current directory instead of treating it as an option.

---

# 🤔 Biggest Confusion

Initially, I kept trying to read files beginning with `-` using the normal `cat` command, which caused Linux to interpret the filename as an option instead of a file. After experimenting with different approaches and observing the error messages, I understood why `--` and `./` solve the problem and when each method should be used.
