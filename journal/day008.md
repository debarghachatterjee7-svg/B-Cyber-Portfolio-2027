# Mistakes Made

My first instinct was to run:

```bash
cat data.txt
```

Since the file contains millions of lines, the terminal became flooded with output and I lost the progress from my previous practice session.

I realized that opening huge files directly is inefficient. The correct approach is to search only for the required information using grep instead of printing the entire file.

---

# Biggest Learning

Choosing the correct command is more important than simply opening a file. grep allows targeted searching, making it much faster and safer when working with very large files.
