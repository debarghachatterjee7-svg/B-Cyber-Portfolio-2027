# 🧠 Important Concepts Learned

Today I finally understood:

### Identity Files

The `-i` flag explicitly tells SSH which private key should be used during authentication.

Rather than searching for default keys inside `.ssh`, OpenSSH directly loads the supplied identity file.

---

### Public Key Authentication

Authentication is performed without sending the private key to the server.

Instead:

- Server stores Public Key
- Client owns Private Key
- Mathematical verification proves ownership

This concept is one of the most important security mechanisms used across modern infrastructure.

---

### Localhost Authentication

Bandit13 introduced the concept that SSH connections do not always occur between two different machines.

A user can authenticate to another account on the **same machine** using:

```
localhost
```

This reinforced my understanding of local authentication workflows.

---

These logs provide valuable insight when troubleshooting authentication failures.
