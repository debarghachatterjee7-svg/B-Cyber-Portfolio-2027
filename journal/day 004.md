## Experiment
While solving Bandit Level 3, I intentionally tried multiple incorrect command variations before finding the correct syntax. This helped me understand how Linux interprets filenames and hidden files.

---

## Observation
- `ls` did not display hidden files.
- `ls -a` revealed the hidden file `...Hiding-From-You`.
- Linux treated `Hiding-From-You` and `...Hiding-From-You` as completely different filenames.
- Incorrect command syntax or missing dots resulted in "No such file or directory" errors.
- The correct command was:

```bash
cat ...Hiding-From-You
```

because the filename itself started with three dots.

---

## Biggest Learning
Linux treats every character in a filename as significant. Hidden files are simply files whose names begin with a dot (`.`), and commands must match the filename exactly.

---

## Biggest Confusion
Initially, I assumed Linux would automatically interpret the hidden filename regardless of the leading dots. After experimenting with several incorrect commands and observing the error messages, I understood that Linux requires the exact filename.
