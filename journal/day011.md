# Mistakes Made

### Mistake 1

Initially wrote:

```bash
base64 - d
```

Linux interpreted `d` as an extra argument.

---

### Mistake 2

Then wrote:

```bash
base64-d
```

Linux searched for a command named `base64-d`, which does not exist.

---

### Solution

Correct syntax:

```bash
base64 -d
```

---

# Biggest Learning

Small syntax mistakes can completely change how Linux interprets a command.

Reading terminal error messages carefully often provides enough information to correct the mistake without searching online.
